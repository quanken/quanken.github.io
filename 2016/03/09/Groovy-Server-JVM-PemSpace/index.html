Groovy Server JVM PermGen space

Mar 09, 2016 in software, java, groovy

在使用Groovy作为开发语言完成服务端程序开发时，服务端在运行了一定时间后，会出现 java.lang.RuntimeException: PermGen space 的异常。

环境

服务端或者开发机器使用的JDK为Oracle JDK 或者 OpenJDK， 大版本为1.7
应用容器采用的是 Apache Tomcat 7
Groovy 版本为 2.3 (其他版本没有实际测试，但从问题原因分析，应该类似)

原因

PermGen Space 全称 Permanent Generation space，在实际应用中，主要被Tomcat 或者其他 J2EE 容器用来存放对象的类定义以及相关的Meta信息（对象的实例存放在Heap）。 当你的程序里面出现大量新的类的加载时，PermGen space 的内容就会快速增长，最后导致溢出。一般这种大量的新类被装载大多数都发生在

  • 开发环境中的热部署
  • JSP预编译
  • 程序大量使用代理动态产生动态对象

对于我们的场景来说，由于是测试服务器不存在热部署，也不存在调整JSP而需要大量预编译深圳JSP的预编译都是很少的，因此基本可以确定是Groovy语言特性所导致。

思路

在这种可能出现大量Class写入到PermGen space中且这些class未必是长久有效的情况，可以通过两种方式来应对：

  • 加大 PermGen 大小

可以在JAVA_OPTS中增加 -XX:MaxPermSize=SIZE , 其中SIZE为大小，可以为 128M 或者其他你评估、测试后可以正常运行的大小。

  • 卸载掉多余的class

可以在JAVA_OPTS中增加 -XX:+CMSClassUnloadingEnabled -XX:+UseConcMarkSweepGC

有些博客或者站点推荐使用 -XX:+CMSPermGenSweepingEnabled ，但从 REDHAT:”Please use CMSClassUnloadingEnabled in place of CMSPermGenSweepingEnabled in the future” message when starting JBoss EAP 文章来看，为了后续更好的兼容性，还是建议使用上面说的配置方式。
